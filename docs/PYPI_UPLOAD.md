# Upload to PyPI - Step by Step Guide

## Prerequisites

1. âœ… Package built successfully (`python3 -m build`)
2. âœ… Tested on TestPyPI
3. âœ… PyPI account created (if not, go to https://pypi.org/account/register/)

## Step 1: Get Your PyPI API Token

1. Go to https://pypi.org/manage/account/token/
2. Click "Add API token"
3. Give it a name (e.g., "fx-sdk-upload")
4. **Scope**: Select "Entire account" (or "Upload packages" if you prefer limited scope)
5. Click "Add token"
6. **IMPORTANT**: Copy the token immediately - it starts with `pypi-` and you won't see it again!

## Step 2: Upload to PyPI

### Option A: Using Environment Variables (Recommended)

```bash
# Set your credentials
export TWINE_USERNAME=__token__
export TWINE_PASSWORD=pypi-YourActualPyPIAPITokenHere

# Upload to PyPI
python3 -m twine upload dist/*
```

### Option B: Using .pypirc File

Create or edit `~/.pypirc`:

```ini
[distutils]
index-servers = pypi

[pypi]
username = __token__
password = pypi-YourActualPyPIAPITokenHere
```

Then upload:
```bash
python3 -m twine upload dist/*
```

### Option C: Interactive (Less Secure)

```bash
python3 -m twine upload dist/*
# Enter username: __token__
# Enter password: pypi-YourActualPyPIAPITokenHere
```

## Troubleshooting

**403 Forbidden:**
- Check token starts with `pypi-`
- Verify username is `__token__` (not your PyPI username)
- Ensure token has upload permissions

**400 Bad Request:**
- Package name might already be taken
- Version might already exist (can't re-upload)
- Check package metadata for errors

**Network errors:**
- Check internet connection
- Try again after a few minutes
- Use `--verbose` flag for more details
